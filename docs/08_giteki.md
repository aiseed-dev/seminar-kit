# 無線通信機器の認証制度とセキュリティ更新の両立に関する論点整理(私案)

作成: 〔氏名〕 2026年7月
取扱: 意見交換用の素材。固有の企業名・製品名は含めない。

## 1. 要旨

ネットワーク機器の安全性は、脆弱性修正(ソフトウェア更新)の継続を前提と
する。一方、電波法に基づく技術基準適合証明・工事設計認証は設計の固定を
前提として運用されており、両者は構造的な緊張関係にある。この結果、
(1) 修正が提供されない認証済み機器が市場に滞留し、(2) 修正を施した機器の
制度上の地位が不明確となり、(3) 再認証負担が中小事業者・研究開発の参入
障壁となっている。セキュリティ更新を前提とした認証運用の明確化と、
中小・地域向けの認証負担の軽減を検討すべきである。

## 2. 現状

- 無線設備を国内で使用するには、電波法に基づく技術基準適合証明または
  工事設計認証(いわゆる技適)を要する。認証は機器と無線部ファームウェア
  の組合せに実質的に紐づく。
- 登録証明機関における試験・認証の費用は、1機種・1構成あたり数十万円
  規模とされる(公開情報からの推定を含む)。無線特性に影響し得る変更は
  変更手続・再認証の対象となり得るが、ホストOS更新等が認証に影響する
  か否かの運用解釈は必ずしも明文で周知されていない。
- 大手事業者は、認証済み無線モジュールと固定ファームウェアを機器本体の
  OSから分離する構造により、OS更新を継続しつつ認証を維持している
  (実務慣行としての推定)。同構造を持たない中小・個人開発者は、更新の
  たびに再認証費用を負担し得ない。

## 3. 論点

**(1) セキュリティ更新と認証維持の構造的衝突**
無線部ファームウェアの脆弱性は現実に繰り返し発見されている
(例: 2017年の Broadcom 製無線チップの脆弱性〔Broadpwn〕、WPA2 の
プロトコル欠陥〔KRACK〕)。修正の適用は安全性の要請である一方、
「設計の固定」を前提とする認証との整合が制度上明確でない。

**(2) サポート終了機器の滞留**
製造者が更新提供を終了した機器は、認証が有効なまま脆弱性が放置される。
制度上適法な機器が安全でなく、第三者が修正を施した機器が制度上不明確
という逆転が生じ得る。

**(3) 負担の非対称と市場への副作用**
再認証費用を吸収できるのは大規模事業者に限られ、中小・個人の参入が
阻害される。国内で正規に購入できる機器の選択肢が乏しい結果、海外通販
経由で認証未取得機器が流入するという副作用を制度自身が生んでいる。
この流入は既に特殊な輸入経路に限られない。海外市場向け型番の
ワークステーション(無線LANモジュール搭載)が国内大手通販サイトの
通常の売場で販売され、価格比較サイト経由で一般消費者が到達する状況が
確認できる(2026年7月、作成者確認)。無線モジュールの認証の有無を
購入者が判別する手段は事実上なく、リスクのみが末端の利用者に
転嫁されている。
なお、電波法が規律するのは電波の発射であり、認証未取得機器の輸入・
販売自体は原則として規制されていない(基準不適合設備に対する勧告・
公表制度があるにとどまる)。商流の全過程が適法である一方、違法と
なり得るのは使用者のみであり、責任の所在が構造的に逆転している。

## 4. 影響

利用者の安全性低下、国内ネットワーク機器産業の空洞化、オンプレミス回帰
(自営サーバー・自営ネットワーク)需要への国内供給の不足。経済安全保障
(通信機器のサプライチェーン)の観点からも国産の選択肢の乏しさは論点となる。

## 5. 対応の方向性(検討の素材)

1. 無線特性(周波数・空中線電力等)に影響しないソフトウェア更新は認証に
   影響しない旨の、運用解釈の明文化・周知(ガイドライン整備)
2. セキュリティ修正に伴う変更手続の簡素化(事後届出制等)
3. 認証済み無線モジュール活用の指針整備(組込み条件の明確化)
4. 中小・研究開発向けの認証費用支援(補助事業のメニュー化)、公設試験
   研究機関(工業技術センター等)における事前評価環境の整備
5. 技適未取得機器の実験等特例制度(届出制)の周知・活用拡充
6. 参考: EU の無線機器指令(RED)は2025年適用のサイバーセキュリティ
   要件において更新可能性を求める方向にあり、「更新を前提とした制度設計」
   は国際的潮流(参考情報)

## 6. 地域における含意(私見)

セキュリティ更新の継続と認証の維持を製品保守として最初から織り込んだ
国産ネットワーク機器は、現在ほぼ空白の領域である。認証実務の経験を
有する県内製造業と公設試の連携により、補助事業を活用した研究開発として
成立する余地がある。
