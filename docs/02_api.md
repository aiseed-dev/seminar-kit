# 02 API 仕様(seminar)

ベースパス: /api/v1。公開・申込系のみ(事務局は Flet・DB直結で API なし)。
日時 ISO 8601(UTC)。エラー形式: { "detail": "...", "code": "..." }。

## 公開(認証不要)

| メソッド/パス | 内容 |
|---|---|
| GET /courses | 募集中の講座一覧(?category=) |
| GET /courses/{id} | 講座詳細(会場残席の有無〔数は出さない〕含む) |
| GET /qr/c/{id}.png | 講座ページQR(チラシ印刷用。segno 生成) |

| GET /docs/session/{course_id} | 経路1: 様式の一時コピーで OnlyOffice Docs の編集セッションを発行(匿名・JWT 署名つき設定を返す)。編集ページは site/ の1枚に DocsAPI JS を読み込んで表示 |
| POST /docs/callback | Document Server の保存コールバック。届いた xlsx を mailin と同じパーサへ(source='web')。成功で受領メール、不備は編集画面にエラー表示+未処理キュー |

公開 API は上記のみ。

## 申込の受信処理(services/mailin.py)

申込専用アドレスに届くメールを2種類に判別して処理する:
**(a) 様式 xlsx の添付あり** → 経路2として読み取り。
**(b) 添付なし+送信者が事前登録済み(companies に一致)** → 経路3
(簡易メール)として本文から講座・受講者名・参加場所を読み取る。
本文の書式は自由度を持たせ、確実に読めない場合は未処理キューへ
(登録済み企業なので事務局が電話一本で補完できる)。

1. 申込専用アドレスの受信箱を監視(IMAP。数分間隔のポーリング)
2. 添付 xlsx を openpyxl で読み取る(名前付きセルで位置を固定。
   様式バージョンセルで版を判定)
3. 検証: 必須セル・参加場所の妥当性(講座の提供有無)・申込期限・
   会場定員(超過時は会場不可の旨を自動返信し、事務局キューへ)
4. 成功: applications+attendees を登録(申込番号を採番)、companies を
   upsert(初回=事前登録、以後は最新情報に更新)、原本を保存
   (received_file)、**受領メール**(申込番号・講座情報・
   「キャンセルはこのメールに返信」「次回は受講者名だけのメールでOK」)
   を自動送信
5. 失敗(添付なし・読取不能・必須欠落): 定型の自動返信(修正のお願い)+
   事務局アプリの「未処理受信」キューに積む(原文ごと)。
   機械で裁けないものは必ず人に落ちる設計とする

キャンセル・変更はメール返信で依頼し、事務局が Flet から処理する。

## 申込様式(神Excel を機械可読に設計する)

- site/ の生成時に、**講座ごとの様式 xlsx を openpyxl で自動生成**
  (講座名・講座ID・開催日を埋め込み済み。講座ページからダウンロード)
- 設計: 入力セル以外はシート保護、参加場所はドロップダウン(入力規則)、
  受講者は3名分の枠、名前付きセルで機械読み取り位置を固定、
  様式バージョンをセルに記録
- 印刷にも耐えるレイアウトにする(FAX・紙の申込書と同じ様式。
  FAX で届いた分は事務局が Flet の代行入力で登録: source='staff')

## メール(services/mail.py)

| トリガ | 宛先 | 内容 |
|---|---|---|
| 受領(自動読取成功) | 担当者メール | 申込番号・講座情報・「変更・キャンセルはこのメールに返信」 |
| 読取不能・不備 | 送信者 | 修正のお願い(定型)。原文は事務局キューへ |
| キャンセル処理(事務局操作) | 担当者 | キャンセル受付 |
| 一斉送信(事務局操作) | 確定受講者(参加場所で絞り込み可) | 配信URL(meeting_url)・リマインド等。mails に記録 |

## 定期ジョブ

- 申込専用アドレスの受信箱ポーリング(mailin。数分間隔)
- 申込期限(apply_deadline)到来で status を closed に自動変更+静的再生成

## 配信URL(meeting_url)

- 講座作成時に自営 Jitsi のルームURLを自動生成(推測されにくい
  ランダムなルーム名)。移行期は Zoom の URL を手入力してもよい
  (フィールドは中立な URL として扱う)
- **公開 API・静的サイトには含めない**。一斉送信でのみ受講者に届ける

## 静的生成

- 講座の保存・状態変更時に、公開静的サイト(一覧・講座ページ)を再生成し
  Cloudflare Pages/R2 へ反映。個人情報は一切含めない
