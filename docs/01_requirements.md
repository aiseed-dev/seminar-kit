# 01 機能要件(seminar)

## 目的

研修・セミナーの運営(告知→申込→連絡→開催→実績報告)を一つの流れに
する。開催する組織の種類は問わない(企業・団体・自治体・支援機関など)。
現行の WordPress+埋め込みフォーム+手作業のメール・名簿集計を置き換え、
運用の手を減らす。

## ユーザー種別

| 種別 | 説明 | 使う画面 |
|---|---|---|
| 訪問者・申込者 | 申込する企業・団体の担当者。アカウント不要 | 公開静的サイト+申込様式(ブラウザ記入/ダウンロード) |
| 事務局スタッフ | 講座運営。PocketBase ログイン(操作者記録) | 事務局アプリ(Flet・DB直結) |
| admin | 分類・スタッフ管理 | 事務局アプリ(管理メニュー) |

## 機能一覧

### 公開静的サイト(Python 生成・Cloudflare Pages)
- P-01 分類別講座一覧(募集中を上に)・講座ページ(概要・講師・開催要領・
      チラシPDF・受講方法の定型文)
- P-02 講座ページに「申し込む」→ 申込へ(ブラウザ記入/様式ダウンロード)。
      ページQR(チラシ印刷用)
- P-03 開催済み講座のアーカイブ(実績の見える化)

### 申込(規模に応じた三経路)

**経路1: ブラウザで記入(OnlyOffice Docs。何もインストールさせない)**
- C-01 講座ページの「ブラウザで記入して送信」→ 申込様式 xlsx を
      自営の OnlyOffice Docs(Document Server)がブラウザ内で開く
      (一時コピー・匿名セッション)。記入して「送信」→ 保存された
      xlsx が経路2と**同じパーサ**(openpyxl・名前付きセル)で読み取られ
      登録(source='web')。受領メール(申込番号つき)を自動送信。
      **様式 xlsx が唯一のフォーム定義**(独自Webフォームは作らない)
- C-02 会場定員超過・期限切れはパーサの検証で弾き、案内を返す。
      スマホでの表計算記入は不向きのため、スマホには様式ダウンロード
      (経路2)か PC を案内する文言を添える

**経路2: 申込様式 xlsx のメール送付(小規模・一般向け、初回)**
- C-03 講座ページから様式 xlsx をダウンロード→記入→申込専用アドレスへ
      添付送付。mailin が自動読み取り(source='mail')。読めないもの・
      不備は修正依頼の自動返信+事務局の未処理キューへ。
      印刷すれば FAX 申込書(事務局が代行入力、source='staff')

**経路3: 簡易メール(事前登録済みの企業)**
- C-04 **初回申込(経路を問わず)で企業情報が自動的に事前登録される**
      (companies に upsert)。二回目からは、登録済みアドレスから
      「講座名(または申込番号)+受講者名+参加場所」だけの短いメールで
      申込できる(source='quick')。**住所・電話番号を毎回書かせない。**
      送信者アドレスの一致が本人確認を兼ねる
- C-05 変更・キャンセルは受領メールへの返信で依頼し、事務局が処理
- C-06 アンケートは紙(会場・サテライト)+メール返信(オンライン)。
      docs/05 のローカルLLMで処理

**システム外の選択肢: 軽量キット(docs/06)**
- 単発・小規模のイベントは本システムを使わず、AI に生成させた
  案内ページ+神Excel+集計スクリプトで、担当者のPCだけで完結できる
  (DB 不要)。使い分けの指針は docs/06

### 事務局アプリ(Flet / Python・DB直結・内部のみ)
- S-01 講座管理: 作成・編集・チラシPDF添付・募集開始/締切/開催済み。
      保存時に静的サイトを再生成
- S-02 申込一覧: 講座別に 受付済み/キャンセル、参加場所別の集計、
      会場残席の表示。申込番号で検索
- S-03 一斉送信: 講座の確定受講者へメール(宛先を参加場所で絞れる。
      **配信URL(自営Jitsiのルーム。移行期はZoomも可)**の送付・
      リマインド・変更連絡)。送信記録が残る(subject/body/宛先数/
      送信者/日時)。配信URLは公開ページに載せない
- S-03a オンライン配信: 講座作成時に自営 Jitsi のルームURLを自動生成
      (meeting_url)。開催時は事務局が同ルームを開いて配信する。
      Zoomのライセンスと外部依存を段階的に不要にする
- S-04 当日名簿: **xlsx で直接出力**(openpyxl。申込番号・企業・受講者・
      参加場所・チェック欄)。印刷して受付で紙でチェックする。
      タップ入力は行わない。Microsoft 365 は不要(OnlyOffice /
      LibreOffice で開ける)
- S-04a 実績入力: 開催後、出席者数(attendance_count)を講座に入力
- S-05 台帳出力: 年度実績台帳(講座数・申込数・確定数・出席数の年度集計。
      事業報告の基礎資料)を xlsx / CSV で出力
- S-06 アンケート: **紙で実施**(様式は OnlyOffice で作成・印刷)。
      回収後、スキャン/撮影した画像をローカルLLM(Command A+ 等)で
      読み取り・集計・自由記述の要約を行う(docs/05)。
      集計値はスタッフが確認してから事業報告に用いる。
      システムはアンケート機能を持たない
- S-07 スタッフ管理(admin): 追加・担当部署名・権限

## 非機能・原則

- 申込番号は DB 全体の通し番号(年+連番)。挿入時にアプリが採番
- 申込者のアカウント(ログイン)は作らない。事前登録は初回申込の副産物として
  自動で生まれ、送信者アドレスで引き当てる(パスワードを持たせない)
- メール(確認・一斉送信)が生命線。送信は services/mail.py に集約し、
  機関の既存メール環境をリレー利用(docs/04)。確認メールの不達に備え
  再送ボタンを事務局アプリに用意
- 個人情報(企業・受講者)を扱うため、公開 API は申込系の最小限のみ。
  参照系の個人情報 API は作らない(公開 API は講座情報と QR のみ)
- 決済なし。受講料は fee_note の表示のみ(有料講座は請求書等アプリ外)
- WordPress との並行稼働: 講座ページと申込のみ新システムに移し、
  既存記事から新講座ページへリンク。段階的にカテゴリ単位で静的化
