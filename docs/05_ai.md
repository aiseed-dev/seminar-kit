# 05 AI 活用環境(参考・任意)

本システムとは独立に、事務局の業務環境として Anthropic の
**非営利団体向けプラン**(Claude Team / Enterprise の非営利価格。
共同作業スペース・アクセス権限・エンタープライズ水準のセキュリティを含む)
を利用できる。対象は 501(c)(3) 相当の非営利団体で、公益財団・公益社団の
ほか、**非営利型の一般財団・一般社団法人、NPO法人等も審査対象になり得る**
(適格性はパートナーの Goodstack が判定。フォームで数分の検証から
始められる。適用条件・価格は Anthropic の最新情報を確認)。

## 事務局での用途(本システムと補完)

- 講座案内文・チラシ文面の下書き(講師略歴の整理・開催要領の定型化)
- 問い合わせ返信の下書き
- (アンケートはローカルLLM側で処理する。下記)
- 年度実績台帳をもとにした事業報告の下書き

## 運用ルール(導入時に定める)

- 個人情報(申込者・受講者の氏名・連絡先)は入力しない。
  分析はアンケートの匿名回答・集計値を対象とする
- 生成文はスタッフが確認してから使用する

## ローカルLLM(機関内・個人情報を扱える側)

クラウドの Claude には個人情報を入れない運用とする一方、機関内の
ローカルサーバーで動くLLM(Ollama 等の実行環境で、日本語の文書業務に
強い Command A+ などのモデル)を併設すれば、実名を含む
データも建物の中で扱える。

**主用途: 紙アンケートの処理。** 会場で回収した紙アンケートを
スキャン/撮影し、視覚対応のローカルモデルで読み取り→選択式の集計→
自由記述の要約、までを機関内で完結させる(オンライン受講者からの
メール返信も同じく要約対象)。手書きの読み取り精度は導入時に実物で
検証し、集計値は必ずスタッフが原本と突き合わせて確認する。
そのほかの用途: 申込傾向の分析、支援記録の下書き。

**ローカルでのコーディング(集計プログラム等)**: xlsx の集計・名簿づくり
のような定型的な Python スクリプトは、ローカルのコーディングモデル
(North Mini Code 等。Apache 2.0・オープンウェイト・30B-A3B で
ローカル実行向き)+ OpenCode 等のエージェントで、機関内で生成できる。
docs/06 の軽量キット(案内ページ+様式+集計プログラム)の生成も同様。
実名データを扱うコードを、コードごと建物の中で書かせる。
設計判断や仕様書全体の解釈を要する開発はクラウドの上位モデルで行い、
絞られた定型タスクをローカルに割り当てる、という使い分けとする。

**位置づけ: VBA の代替。** かつて現場の担当者が Excel マクロ(VBA)で
作っていた「自分の仕事の小さな自動化」を、ローカルAIが生成する
Python スクリプトで置き換える。VBA と違い、Excel 本体のライセンスも
マクロ有効ブックの警告・ブロックも不要で、コードはブックの外に
独立したファイルとして存在し Forgejo で版管理できる。
書ける人の異動で死ぬ「開かずのマクロ」問題も、AI が書き手になる
ことで解消する。

**構築(North Mini Code)**:
- モデル: North Mini Code(Apache 2.0・オープンウェイト。
  30B 総パラメータ / 3B アクティブの MoE で、ローカル実行向き。
  量子化版〔w4a16 等〕でメモリを大きく節約できる)
- 実行環境: Ollama または vLLM をローカルAI機に導入し、
  OpenCode(OSS のコーディングエージェント)から接続する
- 用途の範囲: xlsx 集計・名簿・変換などの**小規模 Python プログラミング
  専用**とする(コーディング特化モデルであり、長文の読解・設計判断には
  使わない。それらは Command A+ またはクラウド側の役割)
- 導入は段階(機材はメモリ要件を見て見積もる)。導入後、引き継ぎ研修で
  「集計プログラムを生成させる演習」を行う(docs/04)
クラウド(非営利プラン)=一般的な文章作成・分析、ローカル=個人情報を
含む内部作業、という二層で使い分ける。ローカルLLM は GPU/メモリ要件が
あるため、導入時期・機材は別途見積もる(任意・段階導入)。

## 示範としての意味

機構は AI 活用講座・DX セミナーを県内企業に提供する立場にあり、
自組織の研修運営を自ら刷新して見せることが、最も説得力のある教材になる。
