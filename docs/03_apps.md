# 03 画面仕様(seminar)

実装規約は CLAUDE.md に従う。

## 公開静的サイト(Python 生成)
- Python(テンプレート)で courses から生成。分類別一覧(募集中→
  開催済み)、講座ページ(概要/講師/開催要領の表/チラシPDF/受講方法の
  定型文/申込ボタン/ページQR)
- 現行 WordPress の該当記事からは新講座ページへリンク(並行稼働)

## ブラウザ記入(OnlyOffice Docs。経路1)

- 講座ページ「ブラウザで記入して送信」→ 編集ページ(site/ の静的1枚+
  DocsAPI JS)が、backend の発行するセッションで様式の一時コピーを開く
- 様式は経路2と同一物(入力規則・保護・名前付きセルもそのまま効く)。
  「送信」ボタン→ 強制保存 → callback で xlsx がパーサへ
- 匿名利用(ログインなし)。一時コピーは登録成功後に received_file として
  保存し、セッションは破棄
- Document Server は Community Edition 9.4 以降、同時編集の接続上限が
  撤廃された(AGPL のまま追加費用なしで本番の主経路にできる)。
  実際の同時記入数の限界はサーバー性能と帯域で決まるため、導入時に
  想定規模で一度確認する。経路2(メール)・電話・紙の並走は、負荷分散と
  アプリ機停止時の受け皿として引き続き維持する

## 申込様式(xlsx。経路2)

- 様式 xlsx は必要時に openpyxl で自動生成する(**公開サイトには
  置かない**)。講座ページには「ブラウザで記入して送信」と
  電話・紙(チラシの申込書)の案内を置く
- 様式の設計: 入力セル以外はシート保護 / 参加場所・受講者数は
  入力規則(ドロップダウン)/ 名前付きセルで読み取り位置を固定 /
  様式バージョン・講座ID・**発行キー**をセルに埋め込み / 印刷しても
  FAX 申込書として使えるレイアウト
- 記入例シートを2枚目に付ける(高齢の担当者にも迷わせない)

## 事務局アプリ(Flet / Python・DB直結)
- PocketBase ログイン(操作者の記録のため)。backend の models / services
  を import。機関内サーバーで `flet run --web`(LAN / SSHトンネル)
```
左ナビ: ダッシュボード / 講座管理 / 申込一覧 / **未処理受信** /
        一斉送信 / 名簿・実績 / エクスポート / スタッフ(admin)
```
- ダッシュボード: 募集中講座と申込数・会場残席、直近の申込
- 講座管理: 一覧(状態別)・編集フォーム・チラシ添付・状態変更・
  **申込様式 xlsx の出力(配布用。公開しないためここから取り出して渡す)**。
  保存で静的サイト再生成
- 申込一覧: 講座選択→受付済み/キャンセル タブ、参加場所別集計、
  申込番号検索、原本 xlsx を開くリンク(received_file)、事務局メモ、
  キャンセル処理(メール依頼を受けて操作)
- 未処理受信: 自動読み取りに失敗した申込メールのキュー(原文・添付つき)。
  内容を見て代行入力(source='staff')するか、差し戻す。
  FAX・紙の申込書もここから代行入力する
- 企業一覧: 事前登録済みの企業マスタ(companies)。検索・編集・
  申込履歴の参照(「この会社は簡易メールでOK」が一目で分かる)
- 一斉送信: 講座選択→宛先(全員/会場/オンライン/サテライト)→件名・本文
  (配信URLの差し込みボタン)→プレビュー→送信(確認ダイアログ)。
  送信履歴の一覧
- 配信: 講座詳細に「配信ルームを開く」ボタン(meeting_url をブラウザで
  開く。自営 Jitsi。司会・画面共有は Jitsi の標準機能)
- 名簿・実績: 講座選択→「当日名簿を xlsx 出力」ボタン(印刷して受付で
  紙チェック)。開催後は同じ画面で出席者数を入力(実績台帳に反映)
- エクスポート: 年度実績台帳を xlsx・CSV で出力
  (アンケートは紙+ローカルLLM処理のため画面なし。docs/05)
