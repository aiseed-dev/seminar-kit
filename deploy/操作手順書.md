# 事務局アプリ 操作手順書(ドラフト)

対象: 事務局スタッフ・運用を引き継ぐ IT 人材。
講座サイクル(告知 → 申込 → 開催 → 実績)を一巡できることがゴール。
画面は事務局アプリ(ブラウザで開く。LAN 内のみ)。ログインは自分の
アカウントで行う(誰が操作したか記録されるため、共用しない)。

## 1. 講座を作って募集を始める

1. 左メニュー「講座管理」→「新規作成」
2. 講座名・分類・開催日時・申込期限・会場・参加場所(会場/オンライン/
   サテライト)・会場定員・受講料を入力。日時は `2026-09-01 13:30` の形式
3. オンラインがある講座は「配信URL」の**自動生成**を押す
   (このURLは公開されない。一斉送信でのみ受講者に届く)
4. 「保存」→「募集開始」。**保存すると公開サイトが自動で再生成される**
5. 公開サイトの講座ページに、ブラウザ記入の入口と電話・紙の案内が
   自動で付く。ページのQRはチラシ印刷に使える。
   **様式 xlsx と申込アドレスは公開されない**(スパム対策)——様式は
   講座編集画面の「**申込様式を出力(配布用)**」で取り出し、
   チラシへの印刷・郵送・メール送付で個別に渡す(様式には発行キーが
   入っており、返送時に発行物として検証される)
6. 様式には「送信用テキスト」シートがある。**推奨は OnlyOffice(無料の
   Desktop 版可)**——マクロが未記入をチェックしたうえで本文を1セルに
   まとめるので、**コピーしてメール本文に貼るだけ**で申込できる(添付
   不要)。Excel 等でも数式で同じ本文が出る(チェックなし)。
   従来どおりのファイル添付でも受け付ける

## 2. 申込を見る

- 「ダッシュボード」: 募集中講座の申込数・会場残席・直近の申込
- 「申込一覧」: 講座を選ぶと受付済み/キャンセルの一覧。
  参加場所別の人数、申込番号での検索、原本(受信した xlsx)のパス表示
- 申込はメール・ブラウザ記入とも**自動で登録され、受領メール
  (申込番号つき)が自動返信されている**。通常は見るだけでよい
- 受領メールが届いていないと言われたら: 該当の申込の「受領メール再送」

## 3. 未処理受信を片付ける(1日1回確認)

自動で読み取れなかった申込メールがここに溜まる(読めないものは
必ずここに落ちる仕組み。黙って消えることはない)。

1. 「未処理受信」を開き、原文を確認
2. 内容が分かる場合 →「代行入力」画面で登録(受領メールが自動送信される)
   → 元のメールを「処理済みへ」
3. 内容が不明な場合 → 電話等で確認するか「差戻しへ」
4. FAX・紙の申込書は、メールを経由せず「代行入力」から直接登録する

## 4. 変更・キャンセルの依頼が来たら

受領メールへの返信で依頼が来る(未処理受信に入る)。
「申込一覧」で該当を探し「キャンセル」を押す(受付メールが自動送信)。
受講者の追加・変更は、いったんキャンセルして代行入力し直すのが確実。

## 5. 開催前: 接続情報を送る

1. 「一斉送信」→ 講座を選ぶ → 宛先(全員/会場/オンライン/サテライト)
2. 件名・本文を書き、オンラインの回がある場合は「配信URLを挿入」
3. 「送信」→ 確認ダイアログ。送信記録は下の履歴に残る
4. リマインドや会場変更の連絡も同じ手順

## 6. 当日: 名簿

「名簿・実績」→ 講座を選ぶ →「当日名簿を xlsx 出力」→ 印刷して受付で
紙でチェックする(アプリでの出欠入力はしない)。

## 7. 開催後: 実績

1. 「名簿・実績」で出席者数を入力して保存(紙名簿のチェックを数える)
2. 講座管理で状態を「開催済みへ」(公開サイトはアーカイブに移る)
3. 年度末は「エクスポート」から実績台帳(xlsx/CSV)を出力し、
   事業報告の基礎資料にする
4. アンケートは紙で回収 → ローカルLLMで読み取り・集計(docs/05。
   集計値は必ず原本と突き合わせて確認する)

## 8. 障害のとき(一次対応)

- **公開サイトは止まらない**(Cloudflare 配信)。講座の閲覧は
  アプリ機が止まっていても可能
- 申込メールはメールサーバーに溜まり、アプリ機の復帰後に自動で順に
  処理される。**急いで何かする必要はない**
- ブラウザ記入だけはアプリ機に依存する。停止中は電話で受け付ける
  (すでに様式を持っている相手はメールで送ってくれれば復帰後に処理される)
- アプリ機の再起動: `systemctl restart seminar-api seminar-mailin seminar-office`
- 直らないときの連絡先: 〔運用支援の窓口を記入〕

## 9. 日々の確認(朝いちの3分)

1. ダッシュボードを開く(アプリが動いている確認を兼ねる)
2. 未処理受信を見る
3. 締切が近い講座の残席を見る
